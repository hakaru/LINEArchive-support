## 追加レビュー 2026-02-21 (GA4・iOS削除・リダイレクト)

### レビュー対象
- ルート `index.html` の自動リダイレクトJS
- 全12HTMLファイルへのGA4タグ（G-7NHBDYBWEY）追加
- 各言語版 `index.html` からのiOSコンテンツ削除

### レビュー結果
1. **プライバシーポリシーの整合性 (重要度: High)**:
   - 全言語の `privacy.html` において「分析データ（アナリティクス）の収集は行いません」との記述が維持されているが、実際にはGA4タグが導入されており、実態と矛盾している。
   - 法的リスクおよびユーザーの信頼性の観点から修正が必須。
2. **GA4計測の正確性 (重要度: Medium/High)**:
   - ルート `index.html` で、GA4タグがリダイレクトJSより前に配置されている。
   - これにより、リダイレクト発生時にルートと遷移先の両方でPageViewがカウントされる「二重計測」のリスクがある。
   - リダイレクトJSを先頭に移動する等の修正を推奨。
3. **リダイレクトロジック (重要度: Low)**:
   - `navigator.language` に基づくリダイレクトは適切に実装されている。
   - `zh` を一律 `zh-Hant` にリダイレクトしている点は、現在のプロジェクト構成（繁体字のみ）では許容されるが、将来的な簡体字対応時には注意が必要。
4. **iOSコンテンツの削除**:
   - 4言語すべての `index.html` において、Apple Storeバッジ、iOS向け手順、タブ切り替えUI/JSが適切に削除されていることを確認。
   - macOS専用アプリとしての訴求に統一されている。
